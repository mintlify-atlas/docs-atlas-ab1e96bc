DrissionPage is a Python web automation library that controls Chromium-based browsers via the Chrome DevTools Protocol and sends HTTP requests through `requests` — all from a single, unified API. Unlike Selenium, it requires no WebDriver, supports cross-iframe element search without context switching, and lets you operate multiple tabs simultaneously.

## Why DrissionPage?

<CardGroup cols={2}>
  <Card title="No WebDriver" icon="circle-check">
    Connects directly via Chrome DevTools Protocol. No driver downloads, no version mismatches.
  </Card>
  <Card title="Unified browser + HTTP" icon="arrows-left-right">
    Switch between full browser control and lightweight HTTP requests on the same page object.
  </Card>
  <Card title="Cross-iframe search" icon="layer-group">
    Find elements inside iframes directly — no `switch_to_frame()` gymnastics required.
  </Card>
  <Card title="Auto-wait & retry" icon="clock-rotate-left">
    Built-in waiting and automatic retry makes flaky network conditions manageable.
  </Card>
</CardGroup>

## Getting started

<Steps>
  <Step title="Install DrissionPage">
    ```bash
    pip install DrissionPage
    ```
  </Step>
  <Step title="Control a browser">
    ```python
    from DrissionPage import ChromiumPage

    page = ChromiumPage()
    page.get('https://example.com')
    print(page.title)
    ```
  </Step>
  <Step title="Find and interact with elements">
    ```python
    # Click a button by text content
    page.ele('text:Submit').click()

    # Fill an input by CSS selector
    page.ele('#username').input('my_user')
    ```
  </Step>
  <Step title="Or send HTTP requests">
    ```python
    from DrissionPage import SessionPage

    page = SessionPage()
    page.get('https://httpbin.org/get')
    print(page.json)
    ```
  </Step>
</Steps>

<Tip>
  Use `WebPage` when you need to switch between browser mode and session mode on the same site — great for bypassing login pages while collecting data efficiently.
</Tip>
