DrissionPage is a Python web automation library that drives Chromium-based browsers directly via the Chrome DevTools Protocol and sends HTTP requests through `requests` — all from a single, unified API. Starting from version 3.0, it no longer depends on Selenium or WebDriver, giving you faster runtime performance, fewer setup headaches, and more powerful element-finding capabilities.

## Key differentiators from Selenium

| Capability | DrissionPage | Selenium |
|---|---|---|
| WebDriver required | No | Yes |
| Driver version management | Not needed | Required per browser version |
| Cross-iframe element search | Direct, no switching | Must `switch_to.frame()` |
| Multiple tabs simultaneously | Yes, without switching | One active tab at a time |
| Full-page screenshots | Yes (Chrome 90+) | Viewport only |
| Shadow DOM (closed) | Supported | Not supported |
| HTTP request mode | Built-in | External library needed |

## Supported environments

- **Python**: 3.6 and above
- **Operating systems**: Windows, Linux, macOS
- **Browsers**: Chromium-based browsers (Chrome, Edge) and Electron applications

## How it works

All three page objects follow the same operational pattern: create a page object, find elements on the page, then read or act on those elements. The element locator syntax is consistent whether you are using browser mode or session mode.

```python
from DrissionPage import ChromiumPage

page = ChromiumPage()
page.get('https://example.com')

# Locate by CSS selector shorthand
page.ele('#submit-btn').click()

# Locate by text content
page.ele('text:Sign in').click()

# Locate by attribute
page.ele('@placeholder=Search').input('DrissionPage')
```

DrissionPage's locator syntax is more concise than XPath or CSS selectors, and it is compatible with both XPath and CSS selector strings when you need them.

## Configuration management

DrissionPage stores browser and session settings in an INI configuration file that is read automatically at startup. You rarely need to write configuration boilerplate in your scripts. When you do need to customize settings — browser executable path, user data directory, proxy, headers — you use `ChromiumOptions` or `SessionOptions` and optionally persist them with `.save()`.
